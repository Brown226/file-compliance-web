# 文件智能审查系统 - 内网快速部署方案

> **原则**：忽略复杂因素，以最短时间实现上线为第一原则
> **适用场景**：内网服务器 80/3000/5432/6379/8000 端口可用

---

## 一、部署前检查清单

### 1.1 环境要求

| 项目 | 要求 | 检查命令 |
|------|------|----------|
| Docker | 20.10+ | `docker --version` |
| Docker Compose | 2.0+ | `docker-compose --version` |
| 磁盘空间 | ≥10GB | `df -h` |
| 内存 | ≥8GB | `free -h` |
| 端口 | 80, 3000, 5432, 6379, 8000 未占用 | `netstat -an \| grep -E ":(80\|3000\|5432\|6379\|8000)"` |

### 1.2 快速预检脚本

```powershell
# Windows PowerShell 执行
docker --version
docker-compose --version
netstat -an | Select-String ":(80|3000|5432|6379|8000)"
```

## 二、镜像加载（内网环境必须）

### 2.1 加载全部镜像（约 1.5GB）

```powershell
# 进入离线包目录
cd offline-packages

# 执行镜像加载（Windows）
```

## 三、快速部署（三选一）

### 方案A：Windows 一键部署（推荐）

```powershell
# 进入离线包目录
cd offline-packages

# 直接执行部署脚本
.\deploy.bat
```

**等待约 30 秒，自动完成以下操作：**
1. 创建 Docker 网络
2. 启动 PostgreSQL
3. 启动 Redis
4. 启动 MarkItDown
5. 启动后端 API
6. 启动前端 Nginx

---

### 方案B：Linux/macOS 一键部署

```bash
cd offline-packages

chmod +x deploy.sh
./deploy.sh
```

---

### 方案C：Docker Compose 部署（最简）

```powershell
# 方式1：使用离线 compose 文件
cd offline-packages/compose-files
docker-compose -f docker-compose.offline.yml up -d

# 方式2：使用完整 compose（含 MaxKB）
docker-compose -f docker-compose.complete.yml up -d
```

## 六、首次使用配置

### 6.1 初始化数据库表结构

容器启动时会自动执行 `init-db.sql`，如果需要手动初始化：

```bash
# 进入后端容器执行 Prisma 迁移
docker exec -it file_review_backend npx prisma db push

# 初始化种子数据
docker exec -it file_review_backend npx tsx src/seed.ts
```

### 6.2 配置 LLM API（系统管理-大模型配置）

首次登录后需要配置 AI 审核能力：

1. 进入「系统管理」→「大模型配置」
2. 添加 LLM 提供商：
   - **API 类型**：OpenAI Compatible
   - **Base URL**：填写内网 LLM 地址（如 http://llm-server:8000/v1）
   - **API Key**：填写密钥
   - **模型名称**：如 `qwen2.5-14b`

### 6.3 配置 OCR（可选）

如需 PDF 扫描件 OCR 能力：

1. 进入「系统管理」→「OCR 配置」
2. 填写 PaddleOCR-VL 内网地址

## 七、常用运维命令

### 7.1 查看日志

```bash
# 查看所有服务日志
docker-compose -f offline-packages/compose-files/docker-compose.offline.yml logs -f

# 查看指定服务日志
docker logs file_review_backend -f
docker logs markitdown_service -f

# 最近 100 行
docker logs --tail 100 file_review_backend
```

### 7.2 重启服务

```bash
# 重启单个服务
docker restart file_review_backend

# 重启全部服务
docker-compose -f offline-packages/compose-files/docker-compose.offline.yml restart
```

### 7.3 停止服务

```bash
# 停止所有服务（保留数据）
docker-compose -f offline-packages/compose-files/docker-compose.offline.yml stop

# 完全停止并移除容器
docker-compose -f offline-packages/compose-files/docker-compose.offline.yml down

# 完全清除（慎用！会删除所有数据）
docker-compose -f offline-packages/compose-files/docker-compose.offline.yml down -v
```

### 7.4 数据备份

```bash
# 备份数据库
docker exec file_review_postgres pg_dump -U file_review_user file_review_db > backup_$(date +%Y%m%d).sql

# 备份上传文件
docker cp file_review_backend:/app/uploads ./uploads_backup

# 查看数据卷
docker volume ls | grep file_review
```

## 八、故障快速排查

### 8.1 容器启动失败

```bash
# 查看容器详情
docker inspect file_review_backend

# 查看启动日志
docker logs file_review_backend
```

### 8.2 数据库连接失败

```bash
# 检查 postgres 是否就绪
docker exec file_review_postgres pg_isready -U file_review_user

# 测试连接
docker exec -it file_review_postgres psql -U file_review_user -d file_review_db
```

### 8.3 文档解析失败

```bash
# 检查 MarkItDown 日志
docker logs markitdown_service

# 本地测试解析
curl -X POST http://localhost:8000/parse \
  -F "file=@/path/to/test.docx"
```

### 8.4 前端无法访问

```bash
# 检查 Nginx 日志
docker logs file_review_frontend

# 测试内部网络
docker exec file_review_frontend curl http://backend:3000/health
```

### 8.5 端口被占用

```powershell
# Windows 查找占用端口的进程
netstat -ano | findstr :3000
taskkill /PID <进程ID> /F
```

## 十、紧急回滚

如部署失败需要回滚到之前状态：

```bash
# 1. 停止所有服务
docker-compose -f offline-packages/compose-files/docker-compose.offline.yml down

# 2. 清理数据卷（谨慎！）
docker volume prune

# 3. 从备份恢复数据库
docker exec -i file_review_postgres psql -U file_review_user -d file_review_db < backup_YYYYMMDD.sql

# 4. 重新部署
./deploy.bat
```
