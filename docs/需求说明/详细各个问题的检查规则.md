# 文件合规检测工具 - 详细检查规则

## 1. 文件命名规范检查

### 1.1 概述
本规则用于检查工程文件的命名是否符合公司移交要求。

### 1.2 图纸文件命名规范

#### 1.2.1 基本格式
```
[项目编码]-[系统编码][类型标识]([版本号])
```

#### 1.2.2 格式详解

| 组成部分 | 格式要求 | 示例 |
|---------|---------|------|
| 项目编码 | 2位字母 + 2位数字 + 1位字母 + 2位数字 + 2位字母 | FJ24A00AC |
| 系统编码 | 3位字母 + 2位数字 | JPS02 |
| 类型标识 | 见下表 | CM、TM、001 等 |
| 版本号 | 括号内单个大写字母 | (A)、(B)、(C) |

#### 1.2.3 图纸类型标识

| 类型标识 | 说明 | 示例 |
|---------|------|------|
| 无（总目录/图册） | 仅项目编码+系统编码+版本 | FJ24A00AC-JPS02(A) |
| 三位数字 | 具体图纸序号 | FJ24A00AC-JPS02-001(A) |
| CM | 封面图（Cover Map） | FJ24A00AC-JPS02-CM(A) |
| TM | 总图/目录图（Title Map） | FJ24A00AC-JPS02-TM(A) |
| FM | 流程图（Flow Map） | FJ24A00AC-JPS02-FM(A) |
| SM | 系统图（System Map） | FJ24A00AC-JPS02-SM(A) |

#### 1.2.4 正则表达式规则

**图册/总目录格式：**
```regex
^[A-Z]{2}\d{2}[A-Z]\d{2}[A-Z]{2}-[A-Z]{3}\d{2}\([A-Z]\)$
```

**具体图纸格式（带序号）：**
```regex
^[A-Z]{2}\d{2}[A-Z]\d{2}[A-Z]{2}-[A-Z]{3}\d{2}-\d{3}\([A-Z]\)$
```

**类型图纸格式（CM/TM/FM/SM）：**
```regex
^[A-Z]{2}\d{2}[A-Z]\d{2}[A-Z]{2}-[A-Z]{3}\d{2}-(CM|TM|FM|SM)\([A-Z]\)$
```

### 1.3 文件（文档）命名规范

#### 1.3.1 基本格式
```
[项目编码]-[系统编码][文件类型][序号]
```

或带版本号：
```
[项目编码]-[系统编码]([版本号])
```

#### 1.3.2 格式详解

| 组成部分 | 格式要求 | 示例 |
|---------|---------|------|
| 项目编码 | 同图纸规范 | FJ24A00AC |
| 系统编码 | 3位字母 + 2位数字 | JPS01、JPS02 |
| 文件类型 | 3位字母 | AJK、APD 等 |
| 序号 | 3位数字 | 001、002 |
| 版本号 | 括号内单个大写字母 | (A)、(B) |

#### 1.3.3 文件类型标识

| 类型标识 | 说明 | 示例 |
|---------|------|------|
| AJK | 安装技术文件 | FJ24A00AC-JPS01AJK001 |
| APD | 设备技术文件 | FJ24A00AC-JPS01APD001 |

#### 1.3.4 正则表达式规则

**带版本号文件：**
```regex
^[A-Z]{2}\d{2}[A-Z]\d{2}[A-Z]{2}-[A-Z]{3}\d{2}\([A-Z]\)$
```

**具体文件格式（带类型和序号）：**
```regex
^[A-Z]{2}\d{2}[A-Z]\d{2}[A-Z]{2}-[A-Z]{3}\d{2}(AJK|APD)\d{3}$
```

### 1.4 检查规则总结

#### 1.4.1 禁止项
- ❌ 文件名中包含中文字符
- ❌ 文件名中包含空格
- ❌ 文件名中包含特殊字符（除 - 和 () 外）
- ❌ 文件名中包含描述性文字（如"的最后一页需要重新打印"）
- ❌ 仅包含中文名称（如"两卡文件"）

#### 1.4.2 允许的文件扩展名
- `.pdf` - PDF文件
- `.doc` / `.docx` - Word文档
- `.xls` / `.xlsx` - Excel表格
- `.dwg` - CAD图纸
- `.txt` - 文本文件（仅允许特定情况）

#### 1.4.3 检查逻辑流程

```
1. 检查文件扩展名是否在允许列表中
   ↓ 否 → 报错：不支持的文件类型
   ↓ 是
2. 检查文件名是否包含中文字符
   ↓ 是 → 报错：文件名不能包含中文
   ↓ 否
3. 检查文件名是否包含空格
   ↓ 是 → 报错：文件名不能包含空格
   ↓ 否
4. 根据文件类型选择对应的正则表达式进行匹配
   ↓ 不匹配 → 报错：命名格式不符合规范
   ↓ 匹配 → 通过
```

#### 1.4.4 错误类型定义

| 错误代码 | 错误描述 | 修复建议 |
|---------|---------|---------|
| NAME_001 | 文件名包含中文字符 | 移除所有中文字符，使用规范编码 |
| NAME_002 | 文件名包含空格 | 移除空格或使用连字符-替代 |
| NAME_003 | 文件名包含非法特殊字符 | 仅保留字母、数字、连字符和括号 |
| NAME_004 | 项目编码格式错误 | 检查项目编码是否为2位字母+2位数字+1位字母+2位数字+2位字母 |
| NAME_005 | 系统编码格式错误 | 检查系统编码是否为3位字母+2位数字 |
| NAME_006 | 图纸类型标识错误 | 检查类型标识是否为CM/TM/FM/SM或三位数字 |
| NAME_007 | 文件类型标识错误 | 检查文件类型是否为AJK/APD等规范类型 |
| NAME_008 | 序号格式错误 | 序号必须为3位数字（001-999） |
| NAME_009 | 版本号格式错误 | 版本号必须为括号内单个大写字母，如(A)、(B) |
| NAME_010 | 文件扩展名不支持 | 使用支持的文件格式（pdf/doc/docx/xls/xlsx/dwg） |

## 4. 内部编码与外部编码一致性检查

### 4.1 检查概述

本规则用于检查 PDF 文件页眉中的编码是否与文件名中的外部编码一致。

### 4.2 问题描述

- **错误情况**：页眉（左上角）使用内部编码标识
- **正确情况**：页眉（左上角）应使用与文件名一致的外部编码

### 4.3 编码定义

| 编码类型 | 格式说明 | 示例 | 出现位置 |
|---------|---------|------|---------|
| 外部编码 | 文件名标准格式 | `FJ24A00AC-JPS02(A)` | 文件名 |
| 内部编码 | 内部管理编码 | `BW610020250053` | 不应出现在页眉 |

### 4.4 检查逻辑

```
1. 从文件名中提取外部编码（不含扩展名）
   示例：FJ24A00AC-JPS02(A).pdf → FJ24A00AC-JPS02(A)

2. 读取 PDF 文件页眉内容（通常位于左上角）

3. 检查页眉中的编码是否与外部编码一致
   - 页眉包含外部编码 → 通过
   - 页眉包含内部编码或不匹配 → 报错

4. 检查页眉中是否包含内部编码格式
   - 内部编码格式：2位字母 + 8位数字 + 6位数字
   - 如：BW610020250053
```

### 4.5 编码匹配规则

#### 4.5.1 外部编码识别

从文件名中提取外部编码的正则表达式：
```regex
^([A-Z]{2}\d{2}[A-Z]\d{2}[A-Z]{2}-[A-Z]{3}\d{2}(?:\([A-Z]\)|-\d{3}\([A-Z]\)|-(?:CM|TM|FM|SM)\([A-Z]\)))$
```

#### 4.5.2 内部编码识别

检测页眉中是否包含内部编码：
```regex
^[A-Z]{2}\d{14}$
```

#### 4.5.3 匹配规则

```
匹配优先级：
1. 精确匹配：页眉内容 == 外部编码（完全一致）
2. 部分匹配：页眉包含外部编码
3. 内部编码检测：页眉包含内部编码格式（错误）
```

### 4.6 错误类型定义

| 错误代码 | 错误描述 | 修复建议 |
|---------|---------|---------|
| CODE_001 | 页眉编码与文件名外部编码不一致 | 修改页眉，确保与文件名中的外部编码一致 |
| CODE_002 | 页眉使用了内部编码，而非外部编码 | 将页眉编码替换为外部编码格式 |
| CODE_003 | 页眉为空或无法识别编码 | 添加正确的外部编码到页眉 |
| CODE_004 | 文件名中无法提取外部编码 | 检查文件名是否符合命名规范 |
| CODE_005 | PDF 无法读取页眉内容 | 检查 PDF 文件是否可正常读取 |

### 4.7 实现说明

#### 4.7.1 PDF 页眉读取方式

```
推荐工具：
- PyPDF2：读取 PDF 页面文本
- pdfplumber：提取页面文本内容
- PyMuPDF (fitz)：快速文本提取

页眉区域定位：
- 通常位于页面顶部 50-100 像素区域
- 需要考虑不同 PDF 模版的布局差异
```

#### 4.7.2 检查流程图

```
开始检查
    ↓
提取文件名中的外部编码
    ↓
读取 PDF 页眉内容（取页眉区域文本）
    ↓
页眉是否为空？
    ├─ 是 → 报错 CODE_003
    └─ 否 → 继续
    ↓
页眉是否包含内部编码格式？
    ├─ 是 → 报错 CODE_002
    └─ 否 → 继续
    ↓
页眉是否与外部编码匹配？
    ├─ 否 → 报错 CODE_001
    └─ 是 → 通过
    ↓
结束
```

## 4.8 封面机组号一致性检查

### 4.8.1 问题描述

封面中不同位置的编码包含机组号信息，**机组号必须保持一致**。

**错误示例：**
| 位置 | 编码 | 机组号位置 | 机组号 |
|------|------|-----------|--------|
| 图册(文件)编号 | `QS2516**0**ED-JPK01` | 第7个字母 | **0** |
| DOC.NO | `QS**2**51600001B25A44GN` | 第3个字母 | **2** |

**问题**：机组号 `0` ≠ `2`，不一致！

**正确示例：**
| 位置 | 编码 | 机组号位置 | 机组号 |
|------|------|-----------|--------|
| 图册(文件)编号 | `QS2516**2**ED-JPK01` | 第7个字母 | **2** |
| DOC.NO | `QS**2**51600001B25A44GN` | 第3个字母 | **2** |

**正确**：机组号 `2` = `2`，一致！

### 4.8.2 机组号位置定义

| 编码类型 | 机组号位置 | 格式说明 | 示例 |
|---------|-----------|---------|------|
| 图册(文件)编号 | 第7个字符 | [字母][字母][数字][数字][数字]**[数字/字母]**[字母][字母]-[字母][字母][数字][数字] | `QS2516**0**ED-JPK01` → 机组号 `0` |
| DOC.NO | 第3个字符 | [字母][字母]**[数字/字母]**[数字][数字][数字][数字][数字][数字] | `QS**2**51600001B25A44GN` → 机组号 `2` |

### 4.8.3 机组号一致性检查逻辑

```
1. 从封面提取图册(文件)编号
   示例：QS25160ED-JPK01
   提取机组号：位置7 = '0'

2. 从封面提取 DOC.NO
   示例：QS251600001B25A44GN
   提取机组号：位置3 = '2'

3. 对比机组号是否一致
   - 0 ≠ 2 → 报错 UNIT_001
   - 0 = 2 → 通过
```

### 4.8.4 机组号提取正则表达式

**图册(文件)编号中的机组号：**
```regex
^[A-Z]{2}\d{4}([A-Z0-9])[A-Z]{2}-[A-Z]{3}\d{2}$
```
捕获组1即为机组号

**DOC.NO 中的机组号：**
```regex
^[A-Z]{2}([A-Z0-9])\d{6}
```
捕获组1即为机组号

### 4.8.5 错误类型定义

| 错误代码 | 错误描述 | 修复建议 |
|---------|---------|---------|
| UNIT_001 | 封面机组号不一致 | 检查并修正封面机组号，确保图册编号和DOC.NO中的机组号一致 |
| UNIT_002 | 无法从图册编号提取机组号 | 检查图册编号格式是否符合规范 |
| UNIT_003 | 无法从DOC.NO提取机组号 | 检查DOC.NO格式是否符合规范 |
| UNIT_004 | 封面缺少图册(文件)编号 | 在封面填写完整的图册(文件)编号 |
| UNIT_005 | 封面缺少DOC.NO | 在封面填写完整的DOC.NO |

## 6. 封面属性对应检查

### 6.1 检查概述

本规则用于检查 PDF 文件封面各属性字段是否填写完整，以及各属性之间是否逻辑对应。

### 6.2 检查要点

#### 6.2.1 必填字段完整性检查

| 字段名称 | 是否必填 | 检查内容 |
|---------|---------|---------|
| 图册(文件)编号 | 必填 | 格式是否符合规范 |
| 共X册/第X册 | 必填 | 册数是否合理 |
| 版次 | 必填 | 是否为单个大写字母 |
| 状态 | 必填 | 是否为标准状态代码 |
| 工程号 | 必填 | 格式是否正确 |
| 子项号或系统号 | 必填 | 是否为两位数字 |
| 子项或系统名称 | 必填 | 是否为空 |
| 设计阶段 | 必填 | 是否为标准阶段名称 |
| 专业 | 必填 | 是否为标准专业名称 |
| 图册(文件)名称 | 必填 | 是否为空 |

#### 6.2.2 字段间逻辑对应检查

| 检查项 | 逻辑规则 | 示例 |
|--------|---------|------|
| 工程号与图册编号 | 工程号应等于图册编号的前缀部分 | 图册编号 F124H0000-ZHS01 → 工程号 F124H0 |
| 子项号与图册编号 | 图册编号中的系统号应与子项号对应 | 图册编号包含 ZHS01 → 子项号 01 |
| 版次一致性 | 封面版次、页眉版次、文件版本应一致 | 封面版次 B = 页眉版次 B = 文件名 (B) |
| 册数合理性 | 第X册应 ≤ 共X册 | 第1册 ≤ 共1册 |

#### 6.2.3 标准值定义

**状态代码：**
- CFC：正式用于施工
- PRE：初步版
- IFA：用于批准
- IFU：用于信息

**设计阶段：**
- 初步设计
- 施工图设计
- 竣工图设计

**专业名称：**
- 综合
- 建筑
- 结构
- 给排水
- 电气
- 暖通
- 消防
- 工艺

### 6.3 错误类型定义

| 错误代码 | 错误描述 | 修复建议 |
|---------|---------|---------|
| ATTR_001 | 图册(文件)编号为空或格式错误 | 填写正确的图册编号 |
| ATTR_002 | 版次为空或格式错误 | 填写单个大写字母作为版次 |
| ATTR_003 | 状态为空或非标准值 | 使用标准状态代码（CFC/PRE/IFA/IFU） |
| ATTR_004 | 工程号为空或与图册编号不匹配 | 确保工程号与图册编号前缀一致 |
| ATTR_005 | 子项号或系统号为空 | 填写两位数字作为子项号 |
| ATTR_006 | 子项或系统名称为空 | 填写子项或系统的完整名称 |
| ATTR_007 | 设计阶段为空或非标准值 | 使用标准设计阶段名称 |
| ATTR_008 | 专业为空或非标准值 | 使用标准专业名称 |
| ATTR_009 | 图册(文件)名称为空 | 填写完整的图册或文件名称 |
| ATTR_010 | 册数不合理（第X册 > 共X册） | 检查并修正册数 |
| ATTR_011 | 工程号与图册编号前缀不一致 | 修正工程号使其与图册编号匹配 |

## 7. 页眉及连续页码检查

### 7.1 检查概述

本规则用于检查 PDF 文件页眉格式是否正确，以及页码是否连续。

### 7.2 页眉内容一致性检查

#### 7.2.1 问题描述

- **错误情况**：页眉中的图册（文件）名称与封面处的图册（文件）名称**不一致**
- **正确情况**：页眉中的名称应与封面名称**完全一致**

#### 7.2.2 检查逻辑

```
1. 读取封面中的"图册(文件)名称"字段内容
   示例：福清核电打包箱式房室外工程变更 施工图设计说明

2. 读取页眉中的名称部分（通常位于页眉中间）
   示例：福清核电打包箱式房变更设计说明

3. 对比两者是否一致
   - 完全一致 → 通过
   - 部分一致/不一致 → 报错
```

#### 7.2.3 匹配规则

| 匹配类型 | 判断标准 | 处理结果 |
|---------|---------|---------|
| 精确匹配 | 页眉名称 == 封面名称 | 通过 |
| 部分匹配 | 页眉名称包含封面名称主要部分 | 警告，建议核对 |
| 不匹配 | 页眉名称与封面名称差异较大 | 报错 |

#### 7.2.4 错误类型定义

| 错误代码 | 错误描述 | 修复建议 |
|---------|---------|---------|
| HEADER_001 | 页眉名称与封面名称不一致 | 修改页眉，使其与封面图册(文件)名称完全一致 |
| HEADER_002 | 页眉名称为空 | 在页眉中添加图册(文件)名称 |
| HEADER_003 | 页眉中缺少关键信息 | 补充完整名称，如"室外工程"、"施工图"等 |

### 7.3 页眉格式检查

#### 7.3.1 标准页眉格式

页眉应包含以下信息（从左到右）：

```
[图册编号]          [图册名称]          [版次]  [页码]
```

**示例：**
```
F124H0000-ZHS01    福清核电打包箱式房室外工程变更施工图设计说明    版次:B    18/21
```

#### 7.3.2 检查要点

| 检查项 | 要求 | 示例 |
|--------|------|------|
| 图册编号 | 位于页眉左侧，与文件名一致 | F124H0000-ZHS01 |
| 图册名称 | 位于页眉中间，与封面一致 | 福清核电打包箱式房室外工程变更施工图设计说明 |
| 版次 | 位于页眉右侧，格式为"版次:X" | 版次:B |
| 页码 | 位于页眉最右侧，格式为"当前页/总页数" | 18/21 |

### 7.4 连续页码检查

#### 7.4.1 检查逻辑

```
1. 遍历 PDF 所有页面的页码
2. 检查页码是否连续递增
3. 检查总页数是否与最后一页的页码一致
```

#### 7.4.2 错误类型定义

## 8. 图纸扫描完整性检查

### 8.1 检查概述

本规则用于检查 CAD 图纸扫描成 PDF 后，图纸内容是否完整扫描，是否存在内容被截断或缺失的情况。

### 8.2 问题描述

- **错误情况**：CAD 图纸扫描成 PDF 后，图纸内容未扫描完整，边缘被截断
- **正确情况**：图纸内容完整，图框、图签等信息完整可见

### 8.3 检查要点

#### 8.3.1 图框完整性检查

| 检查位置 | 检查内容 | 常见问题 |
|---------|---------|---------|
| 左侧边缘 | 图框左边线是否完整 | 左侧内容被截断 |
| 右侧边缘 | 图框右边线是否完整 | 右侧内容被截断（如示例） |
| 顶部边缘 | 图框顶边线是否完整 | 顶部内容被截断 |
| 底部边缘 | 图框底边线是否完整 | 底部内容被截断 |

#### 8.3.2 图签信息完整性检查

图签中以下信息必须完整可见：

| 信息项 | 说明 | 示例 |
|--------|------|------|
| DOC.NO（文件编号） | 完整的文件编码 | F19NX121001B25B43S |
| 工程号 | 工程项目编号 | F124E3 |
| 子项号 | 子项或系统编号 | NX |
| 图号 | 图纸编号 | F124E39NX1DQ |
| 图名 | 图纸名称 | 1、2号机组热洗衣房增加固定灭火设施电气配电图 |
| 图幅 | 图纸幅面规格 | A3、A4 等 |
| 专业 | 专业名称 | 电气 |
| 比例 | 图纸比例 | 1:100 等 |
| 共X张 | 总张数 | 共1张 |

#### 8.3.3 签字栏完整性检查

| 签字栏项目 | 是否必须完整可见 |
|-----------|----------------|
| 审批者 | 是 |
| 设计主持人 | 是 |
| 审核 | 是 |
| 专业负责人 | 是 |
| 校核 | 是 |
| 设计 | 是 |
| 日期 | 是 |

### 8.4 检查方法

#### 8.4.1 基于页面尺寸的检测

```
1. 获取 PDF 页面尺寸（宽度和高度）
2. 获取图框的理论尺寸（根据图幅标准）
3. 对比实际尺寸与理论尺寸
   - 实际尺寸 >= 理论尺寸 → 可能完整
   - 实际尺寸 < 理论尺寸 → 可能不完整
```

#### 8.4.2 基于内容识别的检测

```
1. 提取 PDF 页面边缘区域的图像
2. 检测边缘是否存在图框线
3. 检测图签关键字段是否可识别
4. 检测右侧/底部是否存在内容截断痕迹
```

#### 8.4.3 图幅标准尺寸参考

| 图幅 | 尺寸（mm） | 尺寸（像素，300dpi） |
|------|-----------|---------------------|
| A0 | 841 × 1189 | 9933 × 14043 |
| A1 | 594 × 841 | 7016 × 9933 |
| A2 | 420 × 594 | 4961 × 7016 |
| A3 | 297 × 420 | 3508 × 4961 |
| A4 | 210 × 297 | 2480 × 3508 |

### 8.5 错误类型定义

| 错误代码 | 错误描述 | 修复建议 |
|---------|---------|---------|
| SCAN_001 | 图纸右侧内容未扫描完整 | 重新扫描图纸，确保右侧内容完整 |
| SCAN_002 | 图纸左侧内容未扫描完整 | 重新扫描图纸，确保左侧内容完整 |
| SCAN_003 | 图纸顶部内容未扫描完整 | 重新扫描图纸，确保顶部内容完整 |
| SCAN_004 | 图纸底部内容未扫描完整 | 重新扫描图纸，确保底部内容完整 |
| SCAN_005 | 图签信息不完整 | 重新扫描图纸，确保图签信息完整可见 |
| SCAN_006 | 文件编号显示不全 | 重新扫描图纸，确保DOC.NO完整 |
| SCAN_007 | 签字栏被截断 | 重新扫描图纸，确保签字栏完整 |
| SCAN_008 | 图框线不完整 | 重新扫描图纸，确保图框四边完整 |

### 8.6 实现说明

#### 8.6.1 检测算法思路

```python
# 伪代码示例
def check_scan_integrity(pdf_path):
    # 1. 读取 PDF 页面
    page = read_pdf_page(pdf_path)
    
    # 2. 检测边缘是否存在空白或截断
    edges = detect_page_edges(page)
    
    # 3. 检测图框线是否完整
    border_lines = detect_border_lines(page)
    
    # 4. 检测图签区域
    title_block = extract_title_block(page)
    
    # 5. 检查关键字段
    required_fields = ['DOC.NO', '工程号', '图号', '图名']
    for field in required_fields:
        if not check_field_visible(title_block, field):
            report_error(f"图签字段缺失: {field}")
    
    # 6. 综合判断
    if not is_border_complete(border_lines):
        report_error("图框不完整")
```

### 8.7 注意事项

1. **扫描分辨率**：建议扫描分辨率不低于 300dpi，以确保内容清晰可读
2. **扫描范围**：扫描时应确保图纸四边留有适当边距，避免内容被截断
3. **图幅识别**：系统应能自动识别图纸图幅（A0/A1/A2/A3/A4），并据此判断完整性
4. **批量检查**：对于批量扫描的图纸，建议抽样检查扫描质量
