## 1. 部门主管权限范围

### 1.1 角色定位

部门主管（MANAGER）是本部门的数据管理员，负责：
- 监督和管理本部门及子部门的审查任务
- 复核AI审查结果，确认问题准确性
- 管理本部门用户信息
- 查看本部门操作审计日志

### 1.2 权限矩阵

| 功能模块 | 权限范围 |
|----------|----------|
| 监控仪表盘 | ✅ 本部门数据 |
| 新建审查任务 | ✅ |
| 任务历史 | ⚠️ 本部门及子部门任务 |
| 任务详情 | ⚠️ 本部门及子部门任务 |
| 标准库 | ✅ 查阅（不能修改） |
| 审查规则 | ✅ 查阅（不能修改） |
| 系统管理 | ⚠️ 仅用户管理（本部门） |
| 审计日志 | ⚠️ 本部门日志 |
| LLM配置 | ❌ 无权限 |
| 审查配置 | ❌ 无权限 |

## 3. 创建审查任务

### 3.1 选择审查模式

部门主管可创建以下类型的审查任务：

| 模式 | 适用场景 | 说明 |
|------|----------|------|
| 📝 全文审查 | 全面检查文档质量 | 逐字检查错别字、语法、格式问题 |
| 📚 库审文 | 对照标准库审查 | 检查文档是否符合企业规范 |
| 🔄 一致性检查 | 多文件比对 | 比对多个版本/文件的表述一致性 |
| ⚙️ 自定义规则 | 专项检查 | 使用预设的正则规则进行特定检查 |

### 3.2 操作步骤

1. 点击左侧 **「新建」**
2. 选择审查模式
3. **上传文件**：点击上传区域或拖拽文件
   - 支持多文件同时上传
   - 支持格式：DOCX、PDF、DWG、XLSX、PPTX、TXT
4. **选择参考标准**（库审文模式下）：
   - 上传参考标准文档，或
   - 从标准库选择已有标准
5. **提交任务**：确认文件列表后点击 **「提交任务」**

### 3.3 任务提交后

系统会自动将任务加入处理队列，AI将自动进行审查分析。处理完成后可在 **「历史」** 中查看结果。

## 4. 管理本部门任务

### 4.1 查看任务列表

1. 点击左侧 **「历史」**
2. 系统自动展示**本部门及子部门**的所有任务
3. 可在搜索框输入文件名、任务ID等快速定位

### 4.2 筛选功能

| 筛选条件 | 说明 |
|----------|------|
| 时间范围 | 选择日期区间 |
| 任务状态 | 待处理 / 处理中 / 已完成 / 处理失败 |
| 提交人 | 按提交人筛选 |
| 审查模式 | 全文审查 / 库审文 / 一致性检查 / 自定义规则 |

### 4.3 任务操作

| 操作 | 说明 |
|------|------|
| **查看详情** | 查看审查结果和问题列表 |
| **重新审查** | 对已完成任务重新提交审查 |
| **删除** | 删除已完成的任务（谨慎操作） |

## 5. 审查结果复核

### 5.1 进入任务详情

1. 在任务列表中找到目标任务
2. 点击 **「查看详情」**

### 5.2 结果页面布局

```
┌─────────────────────────────────────┐
│ 任务信息                           │
│ 文件列表  │  问题统计  │  原文预览  │
│─────────────────────────────────────│
│ 问题列表                            │
│ ├── 问题1  [定位] [忽略]            │
│ ├── 问题2  [定位] [忽略]            │
│ └── ...                             │
└─────────────────────────────────────┘
```

### 5.3 问题类型说明

| 类型 | 含义 | 处理建议 |
|------|------|----------|
| 🔤 TYPO | 错别字/拼写错误 | 确认后通知相关人员修改 |
| 📝 GRAMMAR | 语法错误 | 确认后通知相关人员修改 |
| ❌ VIOLATION | 违反规范 | 重点核实，必须整改 |
| ⚠️ INCONSISTENCY | 表述不一致 | 比对原文确认正确表述 |

### 5.4 复核操作

| 操作 | 功能 |
|------|------|
| **定位** | 跳转原文位置，高亮显示问题所在 |
| **忽略** | 标记为已知误报，不再显示 |
| **导出报告** | 下载PDF/Word格式的审查报告 |

### 5.5 审查报告导出

点击 **「导出报告」**，选择格式：
- **PDF报告**：适合存档和打印
- **Word报告**：适合进一步编辑

报告包含：
- 任务基本信息
- 问题统计概览
- 问题详情列表（含原文截图）
- 处理建议

## 6. 部门成员管理

### 6.1 查看部门成员

1. 进入 **系统管理 → 用户管理**
2. 系统自动显示**本部门及子部门**的所有成员
3. 可按姓名、角色筛选

> **注意**：只能看到自己部门及子部门的人员，不能查看其他部门。

### 6.2 编辑成员信息

点击成员右侧 **「编辑」** 按钮，可修改：
- 真实姓名
- 邮箱地址
- 所属部门（仅限本部门及子部门）
- 角色（只能改为 USER）

> **注意**：不能修改登录账号，也不能提升为 MANAGER 或 ADMIN。

### 6.3 禁用/启用成员

- 点击 **「禁用」**：暂停该成员登录权限
- 点击 **「启用」**：恢复该成员登录权限

> **注意**：不能删除成员账号，如需删除请联系管理员。

## 7. 查看审计日志

### 7.1 查看范围

部门主管只能查看**本部门及子部门**的操作日志。

### 7.2 日志字段

| 字段 | 说明 |
|------|------|
| 操作时间 | 精确到秒 |
| 操作人 | 账号及姓名 |
| 操作类型 | 登录、创建任务、查看详情等 |
| 操作内容 | 具体变更详情 |
| IP地址 | 操作来源IP |

### 7.3 筛选功能

可按以下条件筛选：
- 时间范围
- 操作人
- 操作类型

## 8. 实用技巧

### 8.1 批量操作

- 在历史任务列表中，可勾选多个任务进行批量操作
- 支持批量导出报告

### 8.2 快速定位

- 在任务详情页使用 **Ctrl+F** 搜索关键词
- 使用 **「上一个」「下一个」** 按钮快速导航问题

### 8.3 文件命名建议

建议团队成员使用规范的命名方式，便于管理：
```
项目简称_文档类型_版本号_日期.docx
示例：田湾7号_技术方案_v1.2_20260424.docx
```

### 8.4 任务提交最佳实践

| 建议 | 说明 |
|------|------|
| 分模式提交 | 不同审查需求创建不同任务 |
| 控制文件数量 | 单次任务建议不超过10个文件 |
| 及时查看 | 处理完成后尽快复核结果 |
| 定期归档 | 完成审查后及时导出报告存档 |

## 9. 常见问题

### Q1：为什么看不到其他部门的任务？

部门主管只能查看本部门及子部门的任务，不能查看其他部门的数据。如需跨部门查看，请联系管理员。

### Q2：为什么不能修改审查规则？

审查规则和标准库由管理员统一管理，确保全公司审查标准的一致性。如需调整规则，请向管理员反馈。

### Q3：如何给部门新增成员？

部门主管没有新增用户权限。如需新增成员，请联系管理员，或在管理员提供的导入模板中填写后提交。

### Q4：任务处理失败怎么办？

1. 检查文件格式是否支持
2. 检查文件是否损坏
3. 尝试重新上传文件
4. 如多次失败，联系管理员检查LLM服务状态

### Q5：AI审查结果不准确怎么办？

1. 先确认是否为已知问题类型
2. 对误报问题点击 **「忽略」**
3. 如频繁出现误报，向管理员反馈调整审查规则
