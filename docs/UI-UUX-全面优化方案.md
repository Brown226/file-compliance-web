## 1. 执行摘要

### 当前诊断结论

经过对全部 **18 个视图组件**、**9 个 Composable**、**11 个 API 模块**及**5 个样式文件**的逐行分析，当前系统存在以下核心问题：

| 维度 | 评级 | 核心问题 |
|------|------|----------|
| **视觉一致性** | ⚠️ B | 色彩使用混乱，间距不统一，缺乏设计令牌体系 |
| **信息架构** | ⚠️ B+ | 导航结构合理，但 Dashboard 信息密度不足 |
| **交互反馈** | ❌ C | 加载态/空态/错误态处理粗糙，缺少微交互 |
| **数据展示** | ⚠️ B- | 图表基础可用但缺乏洞察力，表格体验平庸 |
| **表单体验** | ⚠️ B | 步骤式流程良好，但校验反馈和辅助提示不足 |
| **响应式** | ❌ C | 几乎无移动端适配，固定宽度布局 |

### 优化目标

1. **用户体验层面**: 降低认知负荷，减少操作步数，提供即时清晰的反馈
2. **视觉品质层面**: 从"能用的内部工具"升级为"专业的企业级 SaaS"
3. **可维护性层面**: 建立统一的设计令牌和组件规范，降低后续开发成本

## 2. 设计系统重构

### 2.1 色彩体系升级

#### 当前问题
- `variables.css` 中定义了基础 CSS 变量，但实际使用中大量硬编码颜色值（`#409EFF`, `#67C23A`, `#F56C6C`, `#E6A23C` 等）
- 缺少语义化色彩映射（success/warning/danger/info 的使用场景不统一）
- 无深浅色阶，无法表达层次关系

### 3.2 仪表盘 (Dashboard.vue) — 双角色设计

> **设计理念**: 仪表盘是系统的信息中枢，但"领导"和"普通用户"的关注点截然不同。
> 领导关注**全局态势和决策指标**，用户关注**个人工作进度和待办行动**。
> 本方案将仪表盘拆分为两种视图，根据用户角色自动呈现。

#### 当前问题
1. **统计卡片信息密度低**: 只有数字 + 图标 + 标题，缺少趋势方向（环比/同比）、迷你图
2. **图表区缺乏交互**: ECharts 图表是静态的，不支持 hover 查看详情、点击下钻
3. **快捷操作入口不够显眼**: 快捷操作卡片样式偏弱
4. **缺乏"待办提醒"**: 没有待处理任务、即将到期任务的提醒区域
5. **最近任务列表缺失**: 没有"最近审查的任务"快速入口
6. **整体布局偏"平"**: 缺乏层次感和视觉焦点
7. **无角色区分**: 领导和普通用户看到完全相同的信息，领导缺乏全局视角，用户缺乏个人聚焦

#### 3.2.1 领导仪表盘 (Admin/Manager Dashboard)

**核心诉求**: 领导不关心单个文件的具体错别字，而是关心"系统整体运转是否健康"、"哪个部门问题多"、"合规趋势是改善还是恶化"。

**领导关心的核心数据维度**:

| 数据维度 | 为什么领导关心 | 展示形式 |
|----------|---------------|----------|
| **审查总量与趋势** | 评估系统使用率、工作量变化 | KPI 大数字 + 趋势折线图（7/30/90天） |
| **整体合规率** | 评估组织文档质量水平 | 环形进度条 + 百分比 |
| **各部门审查对比** | 识别问题集中的部门，针对性管理 | 横向柱状图（可下钻） |
| **违规类型分布** | 了解最常违反哪些规范，推动培训 | 环形饼图 + TOP5 排行 |
| **严重问题预警** | 严重违规需要领导关注和干预 | 红色告警横幅 + 数量角标 |
| **部门审查效率** | 哪些部门审查耗时异常长，可能流程不畅 | 表格 + 甘特/进度条 |
| **系统健康指标** | 任务积压量、失败率、平均处理时长 | KPI 卡片组 |

#### 3.2.2 用户仪表盘 (User Dashboard)

**核心诉求**: 普通用户最关心的是"我有什么活要干"、"我提交的任务审得怎样了"、"有没有问题需要我处理"。仪表盘应该是**个人工作台**而非全局监控屏。

**用户最关心的数据维度**:

| 数据维度 | 为什么用户关心 | 展示形式 |
|----------|---------------|----------|

### 4.1 加载状态统一

| 场景 | 当前状态 | 目标状态 |
|------|----------|----------|
| **页面首次加载** | 白屏或闪烁 | Skeleton 骨架屏（卡片/表格/图表形态） |

| **表格数据加载** | 表头都不显示 | 表头固定 + 行 skeleton 闪烁 |
| **图表加载** | 空白区域 | 渐现动画 + 加载 spinner 居中 |
| **对话框打开** | 直接出现 | 从中心缩放淡入 (scale 0.95 → 1) |

### 4.2 空状态统一

每个列表/表格/数据区域的空状态必须包含:
1. **插画**: 风格统一的 SVG 插图（至少 3 种: 无数据/搜索无结果/出错了）
2. **主文案**: 清晰说明为什么没有数据
3. **辅助文案**: 用户可以做什么
4. **操作按钮**: 主要引导操作（如"新建第一个任务"、"清除筛选"、"重试"）

```
┌────────────────────────────────────────┐
│                                            │
│         📭 (SVG 插图)                     │
│                                            │
│      还没有任何任务                       │

│                                            │
│      [➕ 创建任务]                         │
│                                            │
└────────────────────────────────────────┘
```

### 4.3 错误处理统一

| 错误类型 | 展示方式 | 用户操作 |
|----------|----------|----------|
| **网络错误** | Toast + 页面内 Alert banner | "重试"按钮 |
| **401 未授权** | 自动跳转登录页 + Toast | 重新登录 |
| **403 无权限** | 页面级提示 (非弹窗) | 联系管理员 |
| **404 资源不存在** | 页面级空状态 | 返回列表 |
| **500 服务器错误** | Alert banner + 详细折叠 | "报告问题"/"重试" |
| **表单验证错误** | 字段下方红色文字 + 第一个错误字段 autofocus | 修正输入 |
| **业务逻辑错误** | Dialog 居中展示 | 确认了解 |

### 4.4 成功反馈统一

- **操作成功**: 轻量 Toast (2秒自动消失)，右上角滑入
- **重要成功** (如任务提交完毕): 页面内短暂 Banner "✅ 任务创建成功！预计 3-5 分钟完成审查"
- **异步任务完成**: 浏览器 Notification (需用户授权) + 页面内通知铃铛 badge +1

### 4.5 响应式断点策略

```
Desktop: > 1024px   → 当前完整布局（最小支持分辨率 1280x720）
```

## 附录 A: 推荐的技术选型补充

| 需求 | 推荐方案 | 备注 |
|------|----------|------|
| Markdown 编辑器 | [md-editor-v3](https://github.com/imzbf/md-editor-v3) | Vue 3 原生，中文生态好 |
| 骨架屏 | `el-skeleton` 手动配置 或 [vue-skeleton-loader](https://github.com/LFDP/vue-skeleton-loader) | 优先使用 Element Plus 内置 |
| 全局搜索 (Cmd+K) | 自建 Command Palette 组件 | 参考 Linear/Notion 风格 |
| 流程图绘制 | [Vue Flow](https://vue-flow.github.io/) 或自定义 SVG | Pipeline 可视化 |
| 拖拽排序 | [vuedraggable](https://github.com/SortableJS/vue.draggable.next) | 标准库文件/文件夹拖拽 |
| 差异对比 | [diff](https://github.com/kpdecker/jsdiff) | 提示词版本对比 |
| 图标库 | [Iconify](https://iconify.design/) (集合了多个图标集) | 替代零散的 Unicode/emoji |

#### 3. WebSocket 实时进度推送
- **实现文件**: 
  - 后端: `backend/src/services/websocket.service.ts`, `backend/src/services/review.service.ts`, `backend/src/index.ts`
  - 前端: `frontend/src/composables/useWebSocket.ts`, `frontend/src/views/TaskDetails/index.vue`
- **改动内容**:
  - 后端 WebSocket 服务基于 `ws` 库，集成到 Express HTTP 服务器同一端口
  - 用户连接时通过 JWT token 认证，支持订阅特定任务的进度更新
  - 审查任务处理过程中自动推送进度：初始化→文件处理→分块进度→完成/失败
  - 任务完成时自动推送通知给创建者
  - 前端 WebSocket composable 支持自动重连、心跳保活、任务订阅/退订
  - TaskDetails 页面集成 WebSocket 实时进度条，替代原有的纯轮询机制
  - Vite 开发服务器配置 WebSocket 代理支持
