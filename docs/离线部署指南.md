## 一、环境要求

### 1.1 硬件要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 4 核 | 8 核+ |
| 内存 | 8 GB | 16 GB+ |
| 磁盘 | 50 GB | 100 GB+ SSD |
| 网络 | 内网互通 | 内网互通 |

### 1.2 软件要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| Docker | 20.10+ | 容器运行时 |
| Docker Compose | 2.0+ | 容器编排 |
| 系统 | CentOS 7+ / Ubuntu 20+ / Windows Server 2019+ | |

### 1.3 端口要求

| 端口 | 服务 | 说明 |
|------|------|------|
| 80 | 前端 Web | 主访问端口 |
| 3000 | 后端 API | API 服务端口 |
| 5432 | PostgreSQL | 数据库端口 |
| 6379 | Redis | 缓存端口 |
| 8000 | MarkItDown | 文档解析服务 |
| 8080 | MaxKB (可选) | 知识库服务 |

## 二、离线包内容

### 2.1 镜像清单

```
offline-packages/
├── pgvector-pg15.tar              # PostgreSQL 15 + 向量扩展 (~200MB)
├── redis-7-alpine.tar             # Redis 7 缓存 (~30MB)
├── markitdown-v1.0.tar            # 文档解析服务 (~500MB)
├── backend-v1.0.tar               # 后端 API 服务 (~400MB)
├── frontend-v1.0.tar              # 前端 Web 应用 (~100MB)
├── maxkb-images/                  # MaxKB 知识库 (可选)
│   ├── maxkb.tar
│   ├── maxkb-postgres.tar
│   └── maxkb-redis.tar
└── compose-files/                 # 部署配置文件
    ├── docker-compose.complete.yml
    ├── docker-compose.offline.yml
    ├── .env.offline
    ├── .env.production
    ├── nginx.conf
    ├── init-db.sql
    └── README.md
```

### 2.2 功能说明

| 镜像 | 功能 | 依赖 |
|------|------|------|
| pgvector:pg15 | PostgreSQL 数据库，支持向量存储 | 无 |
| redis:7-alpine | Redis 缓存服务 | 无 |
| markitdown:v1.0 | DOCX/XLSX/PDF/PPTX 文档解析 | PostgreSQL |
| backend:v1.0 | RESTful API 服务 | PostgreSQL, Redis, MarkItDown |
| frontend:v1.0 | Vue 3 Web 应用 | Backend |
| maxkb (可选) | AI 知识库问答 | PostgreSQL, Redis |

## 三、打包步骤（外网环境）

### 3.1 打包前准备

```bash
# 1. 确认公网环境已安装 Docker
docker --version

# 2. 拉取基础镜像
docker pull pgvector/pgvector:pg15
docker pull redis:7-alpine

# 3. 确认项目目录
cd /path/to/file-compliance-web
```

### 3.2 执行打包脚本

**Linux/macOS:**
```bash
chmod +x offline-package-all.sh
./offline-package-all.sh
```

**Windows (PowerShell):**
```powershell
.\offline-package-all.ps1
```

### 3.3 打包验证

```bash
# 检查生成的文件
ls -la offline-packages/
du -sh offline-packages/*

# 预期输出:
# pgvector-pg15.tar         ~200MB
# redis-7-alpine.tar        ~30MB
# markitdown-v1.0.tar       ~500MB
# backend-v1.0.tar          ~400MB
# frontend-v1.0.tar          ~100MB
# 总计约: 1.2GB
```

## 四、离线部署步骤（内网环境）

### 4.1 文件传输

```bash
# 使用 SCP/SMB/RSYNC 等方式传输
scp -r offline-packages.tar user@内网服务器:/opt/

# 或使用 U 盘拷贝
```

### 4.2 解压离线包

```bash
cd /opt
tar -xzf offline-packages.tar
cd offline-packages
```

### 4.3 加载 Docker 镜像

**方式一: 自动加载（推荐）**
```bash
# Windows
.\load-images.bat

# Linux/macOS
chmod +x load-images.sh
./load-images.sh
```

**方式二: 手动加载**
```bash
# 逐个加载
docker load -i pgvector-pg15.tar
docker load -i redis-7-alpine.tar
docker load -i markitdown-v1.0.tar
docker load -i backend-v1.0.tar
docker load -i frontend-v1.0.tar

# 加载 MaxKB (可选)
docker load -i maxkb-images/maxkb.tar
docker load -i maxkb-images/maxkb-postgres.tar
docker load -i maxkb-images/maxkb-redis.tar

# 验证加载
docker images | grep -E "file-review|pgvector|redis|maxkb"
```

### 4.4 配置环境变量

```bash
cd compose-files

# 复制环境变量文件
cp .env.offline .env

# 编辑配置（根据实际情况修改）
vi .env
```

**关键配置项:**
```env
# 数据库
DATABASE_URL=postgresql://file_review_user:file_review_password@localhost:5432/file_review_db

# Redis
REDIS_URL=redis://localhost:6379

# JWT 密钥（生产环境请修改）
JWT_SECRET=your-secret-key-here

# MarkItDown 服务地址
PARSER_SERVICE_URL=http://localhost:8000
```

### 4.5 创建网络

```bash
docker network create file-review-network 2>/dev/null || echo "Network already exists"
```

### 4.6 启动服务

**方式一: 使用 docker-compose（推荐）**
```bash
# 完整部署（包含所有服务）
docker-compose -f docker-compose.complete.yml up -d

# 基础部署（不含 MaxKB）
docker-compose -f docker-compose.offline.yml up -d

# 查看日志
docker-compose -f docker-compose.complete.yml logs -f

# 查看状态
docker-compose -f docker-compose.complete.yml ps
```

**方式二: 手动启动**
```bash
# 1. 启动数据库
docker run -d \
  --name file_review_postgres \
  -e POSTGRES_USER=file_review_user \
  -e POSTGRES_PASSWORD=file_review_password \
  -e POSTGRES_DB=file_review_db \
  -p 5432:5432 \
  -v file_review_postgres_data:/var/lib/postgresql/data \
  --network file-review-network \
  --restart unless-stopped \
  pgvector/pgvector:pg15

# 2. 启动 Redis
docker run -d \
  --name file_review_redis \
  -p 6379:6379 \
  -v file_review_redis_data:/data \
  --network file-review-network \
  --restart unless-stopped \
  redis:7-alpine redis-server --appendonly yes

# 3. 等待数据库就绪
sleep 15

# 4. 启动 MarkItDown
docker run -d \
  --name markitdown_service \
  -p 8000:8000 \
  -e PORT=8000 \
  --network file-review-network \
  --restart unless-stopped \
  file-review-markitdown:v1.0

# 5. 启动后端
docker run -d \
  --name file_review_backend \
  -p 3000:3000 \
  -e NODE_ENV=production \
  -e DATABASE_URL=postgresql://file_review_user:file_review_password@file_review_postgres:5432/file_review_db \
  -e REDIS_URL=redis://file_review_redis:6379 \
  -e PARSER_SERVICE_URL=http://markitdown_service:8000 \
  -e JWT_SECRET=your-secret-key \
  --network file-review-network \
  --restart unless-stopped \
  file-review-backend:v1.0

# 6. 启动前端
docker run -d \
  --name file_review_frontend \
  -p 80:80 \
  --network file-review-network \
  --restart unless-stopped \
  file-review-frontend:v1.0
```

## 六、MaxKB 知识库集成（可选）

### 6.1 MaxKB 镜像加载

```bash
# 加载 MaxKB 相关镜像
cd maxkb-images
for f in *.tar; do docker load -i "$f"; done
```

### 6.2 修改 docker-compose 配置

编辑 `docker-compose.complete.yml`，取消 MaxKB 相关配置的注释：

```yaml
services:
  # ... 其他服务 ...

  maxkb:
    image: registry.fit2cloud.com/maxkb/maxkb:v2.8.0
    container_name: maxkb
    hostname: maxkb
    ports:
      - "8080:8080"
    volumes:
      - maxkb-logs:/opt/maxkb/logs
      - maxkb-local:/opt/maxkb/local
    env_file:
      - maxkb-runtime/maxkb/conf/maxkb.env
    depends_on:
      - maxkb-postgres
      - maxkb-redis
    restart: unless-stopped
    networks:
      - file-review-network

  maxkb-postgres:
    image: registry.fit2cloud.com/maxkb/postgres:v2.8.0
    container_name: maxkb-pgsql
    # ... 配置同上 ...
```

### 6.3 配置知识库 API

在后端 `.env` 中配置 MaxKB 连接:

```env
MAXKB_ENABLED=true
MAXKB_API_URL=http://localhost:8080/api
MAXKB_API_KEY=your-maxkb-api-key
```

## 七、常见问题排查

### 7.1 数据库连接失败

```bash
# 检查 PostgreSQL 是否正常
docker logs file_review_postgres

# 进入数据库容器测试连接
docker exec -it file_review_postgres psql -U file_review_user -d file_review_db

# 检查网络连通性
docker exec -it file_review_backend ping postgres
```

### 7.2 文档解析失败

```bash
# 检查 MarkItDown 日志
docker logs markitdown_service -f

# 测试 MarkItDown 服务
curl -X POST http://localhost:8000/parse \
  -H "Content-Type: application/json" \
  -d '{"file_path": "/path/to/test.docx"}'
```

### 7.3 前端无法访问后端

```bash
# 检查 Nginx 配置
docker exec file_review_frontend cat /etc/nginx/conf.d/default.conf

# 检查 API 代理
docker exec file_review_frontend curl http://backend:3000/health
```

### 7.4 容器启动失败

```bash
# 查看详细错误
docker-compose -f docker-compose.complete.yml up --no-detach

# 检查镜像是否正确加载
docker images | grep file-review
```

## 八、数据备份与恢复

### 8.1 数据备份

```bash
# 备份 PostgreSQL 数据
docker exec file_review_postgres pg_dump -U file_review_user file_review_db > backup.sql

# 备份 Redis 数据
docker exec file_review_redis redis-cli SAVE
docker cp file_review_redis:/data dump.rdb

# 备份上传文件
tar -czf uploads-backup.tar.gz uploads/
```

### 8.2 数据恢复

```bash
# 恢复 PostgreSQL 数据
docker exec -i file_review_postgres psql -U file_review_user -d file_review_db < backup.sql

# 恢复上传文件
tar -xzf uploads-backup.tar.gz
```
