## 4. 用户管理

### 4.1 新增用户

**操作步骤**：

1. 进入 **系统管理 → 用户管理**
2. 点击右上角 **「新增用户」** 按钮
3. 在弹窗中填写表单：

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 登录账号 | 系统唯一标识（如 `zhangsan`） | ✅ |
| 真实姓名 | 员工真实姓名 | ✅ |
| 登录密码 | 初始密码（建议首次登录后修改） | ✅ |
| 角色 | 选择 USER / MANAGER / ADMIN | ✅ |
| 所属部门 | 选择已创建的部门 | ✅ |
| 邮箱 | 接收通知的邮箱地址 | ❌ |

4. 点击 **「确定」** 保存

### 4.2 批量导入用户

适用于大规模员工初始化，支持一次性导入数百人。

**操作步骤**：

1. 进入 **系统管理 → 用户管理**
2. 点击 **「批量导入」**
3. 点击 **「下载模板」** 获取Excel模板
4. 按模板格式填写用户信息
5. 上传Excel文件
6. 预览数据确认无误后，点击 **「确认导入」**

**导入模板字段说明**：

| 字段 | 说明 | 必填 | 示例 |
|------|------|:----:|------|
| 登录账号 | 唯一标识，建议使用拼音缩写 | ✅ | `zhangsan` |
| 真实姓名 | 员工姓名 | ✅ | 张三 |
| 密码 | 初始密码 | ✅ | 123456 |
| 角色 | USER / MANAGER / ADMIN | ✅ | USER |
| 一级部门 | 顶级部门名称 | ✅ | 总公司 |
| 二级部门 | 子部门名称 | ❌ | 研发部 |
| 三级部门 | 更细分子部门 | ❌ | 前端组 |
| 邮箱 | 通知邮箱 | ❌ | xxx@cnpe.cc |

**部门层级说明**：

系统支持三级部门结构，导入时按路径匹配：
```
总公司
├── 一级部门（如：设计管理部）
│   ├── 二级部门（如：结构组）
│   │   └── 三级部门（如：核电结构组）
```

> **提示**：部门名称在同一父级下必须唯一。如系统中存在同名部门，请使用路径精确匹配。

### 4.3 编辑/禁用/删除用户

| 操作 | 说明 |
|------|------|
| **编辑** | 修改用户姓名、邮箱、部门、角色 |
| **禁用** | 暂停用户登录权限，数据保留 |
| **重置密码** | 恢复为默认密码（123456） |
| **删除** | 永久删除用户账号（需谨慎操作） |

## 5. 部门管理

### 5.1 添加部门

1. 进入 **系统管理 → 部门管理**
2. 点击 **「新增部门」**
3. 选择父级部门（如为顶级部门则选"无"）
4. 输入部门名称
5. 点击 **「确定」**

### 5.2 部门层级结构

核审通支持**三级部门树形结构**：

```
中国核电工程有限公司河北分公司
├── 分公司领导
├── 分公司副总工程师
├── 专家委员会
├── 综合办公室
├── 设计管理部
│   ├── 结构组
│   └── 工艺组
├── 核工程所
├── 核电工艺所
├── 电气自动化所
└── ...
```

### 5.3 修改/删除部门

| 操作 | 说明 |
|------|------|
| **编辑** | 修改部门名称或调整上级部门 |
| **删除** | 删除空部门（含子部门的需先删除子部门） |

> **注意**：已有人员关联的部门不能删除，需先将人员调整到其他部门。

## 6. 标准库与审查规则

### 6.1 标准库管理

标准库是AI审查的依据，管理员负责维护企业标准规范。

#### 添加标准

1. 进入 **「标准库」**
2. 点击 **「新增标准」**
3. 填写信息：
   - 标准编号（如 `GB 50011-2010`）
   - 标准名称
   - 标准内容（支持多段落）
   - 分类标签
4. 保存

#### 标准分类

建议按以下维度分类：
- 设计规范
- 安全标准
- 质量标准
- 文档规范
- 术语标准

### 6.2 审查规则管理

1. 进入 **「审查规则管理」**
2. 配置审查项：
   - 错别字规则
   - 术语规范
   - 格式要求
   - 自定义正则表达式

## 7. LLM服务配置

管理员负责配置AI大语言模型接口，确保审查服务正常运行。

### 7.1 配置参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| API密钥 | 模型服务商提供的密钥 | sk-xxxxxxxx |
| API基础URL | 模型接口地址 | https://api.openai.com/v1 |
| 模型名称 | 使用的模型版本 | gpt-4、qwen-plus |
| 最大Tokens | 单次请求最大token数 | 4096 |
| 温度参数 | 输出随机性（0-1） | 0.7 |
| 超时时间 | 请求超时阈值（秒） | 120 |

### 7.2 配置步骤

1. 进入 **「LLM服务配置」**
2. 填写通用模型配置表单
3. 填写OCR视觉模型配置（用于扫描件识别）
4. 点击 **「测试连接」** 验证可用性
5. 保存配置

### 7.3 配置建议

| 场景 | 推荐配置 |
|------|----------|
| 内网部署 | 使用本地部署的Ollama/VLLM服务 |
| 公网环境 | 可使用火山引擎、硅基流动等商用API |
| 高精度需求 | 选用 GPT-4 / Claude-3.5 级别模型 |
| 成本敏感 | 选用 Qwen / GLM 等国产模型 |

## 9. 审计日志

审计日志记录平台所有操作行为，用于安全追溯。

### 9.1 查看日志

1. 进入 **「审计日志」**
2. 使用筛选条件：
   - 时间范围（开始日期 - 结束日期）
   - 操作人（账号或姓名）
   - 操作类型（登录/新增/修改/删除等）
3. 查看操作详情

### 9.2 日志字段说明

| 字段 | 说明 |
|------|------|
| 操作时间 | 精确到秒的时间戳 |
| 操作人 | 执行操作的账号及姓名 |
| 操作类型 | 如：登录、创建任务、修改用户等 |
| 操作内容 | 具体的变更明细 |
| IP地址 | 操作来源IP |
| 操作结果 | 成功/失败 |

### 9.3 日志导出

支持导出Excel格式，用于合规审计存档。

## 10. 常见问题

### Q1：如何重置管理员密码？

如果忘记管理员密码，需通过数据库直接修改，或联系更高级别的系统维护人员。

### Q2：批量导入失败怎么办？

检查Excel格式：
- 确保使用系统下载的模板格式
- 检查必填字段是否完整
- 检查部门名称是否与系统中一致
- 检查角色值是否为 USER/MANAGER/ADMIN 之一

### Q3：如何查看系统运行状态？

进入 **监控仪表盘**，查看数据库、缓存服务状态指示灯。

### Q4：任务处理超时怎么办？

检查LLM服务配置是否正常，确认AI模型接口可访问。可在LLM配置页面点击「测试连接」验证。

### Q5：如何备份数据？

系统使用PostgreSQL数据库，建议使用 `pg_dump` 工具定期备份。
