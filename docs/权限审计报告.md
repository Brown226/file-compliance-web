## 4. 安全机制分析

### 4.1 认证机制

- **JWT Token 认证**: `backend/src/middlewares/auth.middleware.ts`
- Token 存储于请求头: `Authorization: Bearer <token>`
- 支持 Token 黑名单机制: `TokenService.isTokenBlacklisted()`

### 4.2 审计日志

- **自动记录**: 所有 POST/PUT/DELETE 请求
- **实现位置**: `backend/src/middlewares/audit.middleware.ts`
- **日志表**: `audit_logs`
- **脱敏处理**: 自动过滤 `password` 字段

### 4.3 密码安全

- **密码加密**: bcryptjs (10轮盐值)
- **修改密码**: 需提供原密码验证
- **最小长度**: 6位

## 5. 最小权限原则评估

### 5.1 符合最小权限原则的方面 ✅

1. **数据隔离**: USER 角色仅能访问自己的任务数据
2. **部门隔离**: MANAGER 仅能访问本部门及子部门数据
3. **写操作受限**: 系统配置、规则管理等写操作仅 ADMIN 可执行
4. **审计追溯**: 所有敏感操作均记录审计日志

### 5.2 潜在风险点 ⚠️

| 风险项 | 描述 | 严重程度 | 建议 |
|--------|------|:--------:|------|
| 删除无限制 | 普通用户可删除自己创建的任务 | 中 | 当前设计合理，任务删除后可恢复 |
| 误报标记权限过宽 | 所有用户均可标记/取消误报 | 低 | 建议仅 MANAGER 及以上可取消他人工件的误报 |
| MANAGER 无写权限 | MANAGER 无法管理部门内的标准/规则 | 中 | 如有需求可考虑扩展 |
| API 无速率限制 | 未实现请求频率限制 | 高 | 建议添加 Redis 限流中间件 |

### 5.3 缺失的安全措施 ❌

| 措施 | 状态 | 说明 |
|------|:----:|------|
| API 速率限制 | ❌ | 未实现 |
| 敏感操作二次验证 | ❌ | 未实现 |
| 会话超时自动登出 | ❌ | 未实现 |
| IP 白名单 | ❌ | 未实现 |
| 密码强度策略 | ❌ | 仅要求6位 |
| 账号锁定策略 | ❌ | 未实现 |

## 6. 开发权限说明

### 6.1 关于"开发权限"

本项目是 **Web 应用系统**，其权限设计的是 **运行时业务权限**（用户在系统中能做什么），而非 **代码仓库的开发权限**（如 Git 分支、代码提交等）。

### 6.2 权限配置存储位置

| 配置项 | 文件位置 |
|--------|----------|
| 角色定义 | `backend/prisma/schema.prisma` |
| RBAC 中间件 | `backend/src/middlewares/rbac.middleware.ts` |
| 路由权限 | `backend/src/routes/*.routes.ts` |
| 前端权限 | `frontend/src/stores/user.ts` |
| 路由守卫 | `frontend/src/router/index.ts` |
| 菜单控制 | `frontend/src/components/layout/AppLayout.vue` |

### 6.3 Git 仓库权限（项目级）

此项目的 Git 权限由代码托管平台（如 GitHub/GitLab）管理，不在本系统范围内。建议配置:

| 分支 | 推荐权限 |
|------|----------|
| main | 仅管理员可合并，需 PR Review |
| develop | 开发者可推送，需至少1人 Review |
| feature/* | 开发者可创建和推送 |
| hotfix/* | 仅管理员可合并 |

## 7. 权限变更建议

### 7.1 短期优化

1. **添加 API 速率限制**: 使用 Redis 实现
2. **增强密码策略**: 至少8位，包含大小写字母和数字
3. **添加会话超时**: 建议 30 分钟无操作自动登出

### 7.2 中期扩展

1. **操作二次验证**: 删除/导出等敏感操作需二次确认
2. **细粒度权限**: 支持自定义权限角色
3. **权限变更审批**: 记录权限变更申请和审批流程

### 7.3 长期规划

1. **SSO 集成**: 对接企业 AD/LDAP
2. **权限审计报表**: 自动生成权限合规报告
3. **实时异常告警**: 异常权限使用实时告警
