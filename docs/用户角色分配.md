## 用户角色权限体系分析

### 一、角色定义（三级 RBAC）

系统定义在 `schema.prisma` 中：

```
ADMIN  ──►  MANAGER  ──►  USER
(全局)        (部门)      (个人)
```

| 角色 | 等级 | 核心原则 |
|------|------|---------|
| **ADMIN** | 最高 | 无限制，可管理所有数据 |
| **MANAGER** | 中 | 可访问本部门及所有子部门的数据 |
| **USER** | 最低 | 仅可访问自己创建的数据 |

### 六、实际交互边界示例

**场景1：USER 创建任务后查看**
- 可以看到自己创建的任务
- 看不到其他人创建的任务
- Dashboard 默认进入"我的工作台"

**场景2：MANAGER 查看任务历史**
- 可以看到本部门所有成员的任务
- 可以看到子部门所有成员的任务
- 无法修改任何任务的审查模式配置
- Dashboard 默认进入"领导监控台"（因 `canSwitchView = isAdmin() || isManager()`）

**场景3：USER 尝试访问 `/pipeline-config`**
- 路由守卫 `beforeEach` 检测到 `requiresAdmin: true` 且 `!isAdmin()`
- 自动重定向到 `/dashboard`
- 浏览器 F12 看不到任何网络请求到达后端
