# Web Portal 界面与功能实现 Spec

## Why
当前的文件智能审查系统为“Windows客户端+CAD插件”的单机形态，无法满足企业级多用户并发、标准库集中管理以及脱离 CAD 环境查看审查报告的需求。通过构建全新的企业端 Web Portal，可以实现统一入口、异步批量处理和审查结果的在线可视化，极大提升团队协作效率。此外，作为企业级应用，必须具备完善的账号权限隔离、数据可视化监控以及审计日志能力。

## What Changes
- 构建基于 Vue 3 + TypeScript 的企业级单页应用（SPA）。
- 采用经典的左侧边栏导航布局。
- 新增独立的“新建审查任务”页面，支持批量拖拽上传图纸（DWG）和文档（Word, Excel, PDF, 扫描版 PDF）并配置审查参数。
- 新增基础“任务列表”页面，提供按状态和日期的检索过滤。
- 新增“审查结果展示”页面，采用左右分栏设计（左侧文件树，右侧错误明细），并集成“复制 Handle ID 以供 CAD 定位”功能。
- 新增“企业标准库/项目字典”管理页面，支持基础的增删改查（CRUD）与 Excel 批量导入。
- 新增“导出 Excel”一键下载审查报告功能。
- **新增** 用户登录与个人密码修改功能（无前端开放注册入口，由后台统一开号）。
- **新增** 部门组织架构与账号管理页面（基于 RBAC 的严格数据隔离：员工看自己、主管看部门、管理员看全局）。
- **新增** 业务质量数据分析看板（Dashboard），作为登录后的首屏，展示高频错误和部门审查质量对比。
- **新增** 系统操作审计日志统计页面，记录关键业务动作。

## Impact
- Affected specs: 前端界面交互规范、API 接口定义规范、数据库权限模型。
- Affected code: `frontend/` 目录下的所有视图组件、路由配置及鉴权拦截器（Router Guard）。

## ADDED Requirements
### Requirement: 整体框架布局
系统 SHALL 提供一个带有左侧主导航栏（包含：Dashboard、工作台、新建任务、任务历史、标准库管理、系统管理）和顶部用户信息栏的标准企业级布局。

### Requirement: 企业级登录与账号体系
系统 SHALL 仅允许通过管理员后台创建的账号进行登录。
#### Scenario: 员工首次登录
- **WHEN** 员工拿到管理员分配的初始账号密码，在登录页输入并提交
- **THEN** 登录成功并强制/建议修改初始密码，不允许自己通过手机号或邮箱注册新账号。

### Requirement: 部门与数据严格隔离
系统 SHALL 根据用户的角色和所属部门控制“任务列表”的数据可见性。
#### Scenario: 部门主管查看任务
- **WHEN** 某部门主管进入“任务列表”页
- **THEN** 表格中不仅展示该主管自己上传的任务，还展示本部门所有下属员工发起的历史审查任务，但无法看到其他部门的数据。

### Requirement: 业务质量数据看板（Dashboard）
系统 SHALL 提供一个基于 ECharts 的可视化看板作为首屏。
#### Scenario: 管理层查看全公司出图质量
- **WHEN** 管理员或高管登录系统进入 Dashboard
- **THEN** 页面直观展示：本月“最高频错别字 Top 10”、“最常被违反的标准规范 Top 10”柱状图，以及“各部门平均单图错误率对比”折线/饼图。

### Requirement: 基础操作审计日志
系统 SHALL 提供系统日志查询界面，供管理员审计。
#### Scenario: 追溯标准库修改记录
- **WHEN** 管理员在日志统计页面搜索“标准库修改”动作
- **THEN** 列表清晰展示哪位员工在什么时间、什么 IP 地址删除了某条企业规范。

### Requirement: 任务创建与上传
系统 SHALL 提供独立的“新建任务”页面。
#### Scenario: 批量上传混合类型文件发起审查
- **WHEN** 用户进入新建任务页，将 100 个文件（包括 DWG、Word、Excel、可编辑 PDF 和扫描版 PDF）拖入上传区域，选择适用的“审查标准”并点击“开始审查”
- **THEN** 系统显示上传进度，全部上传完成后，自动跳转至“任务列表”页，并显示该任务状态为“排队中/解析中”。系统底层能根据文件类型自动选择对应的解析引擎（如 OCR 识别扫描版 PDF）。

### Requirement: 任务列表与过滤
系统 SHALL 提供任务历史记录表格。
#### Scenario: 查找历史任务
- **WHEN** 用户在任务列表页选择日期范围和状态（如：已完成）
- **THEN** 表格自动过滤并展示符合条件的审查任务记录。

### Requirement: 审查结果展示（分栏视图）
系统 SHALL 提供左右分栏的审查结果详情页。
#### Scenario: 审查员查看具体文件的错误并准备在 CAD 中修改
- **WHEN** 用户在左侧面板点击某个文件名，并在右侧错误列表中点击某条错误的“在 CAD 中定位”按钮
- **THEN** 系统的剪贴板自动复制了该错误的 Handle ID（如 `2A5F`），并提示“已复制，请在 CAD 中使用快捷键定位”。

### Requirement: 标准库维护
系统 SHALL 提供标准库和字典的管理界面。

## MODIFIED Requirements
无。
