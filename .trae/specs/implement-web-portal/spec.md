# Web Portal 界面与功能实现 Spec

## Why
当前的文件智能审查系统为“Windows客户端+CAD插件”的单机形态，无法满足企业级多用户并发、标准库集中管理以及脱离 CAD 环境查看审查报告的需求。通过构建全新的企业端 Web Portal，可以实现统一入口、异步批量处理和审查结果的在线可视化，极大提升团队协作效率。

## What Changes
- 构建基于 Vue 3 + TypeScript 的企业级单页应用（SPA）。
- 采用经典的左侧边栏导航布局。
- 新增独立的“新建审查任务”页面，支持批量拖拽上传图纸和配置审查参数。
- 新增基础“任务列表”页面，提供按状态和日期的检索过滤。
- 新增“审查结果展示”页面，采用左右分栏设计（左侧文件树，右侧错误明细），并集成“复制 Handle ID 以供 CAD 定位”功能。
- 新增“企业标准库/项目字典”管理页面，支持基础的增删改查（CRUD）与 Excel 批量导入。
- 新增“导出 Excel”一键下载审查报告功能。

## Impact
- Affected specs: 前端界面交互规范、API 接口定义规范。
- Affected code: `frontend/` 目录下的所有视图组件和路由配置。

## ADDED Requirements
### Requirement: 整体框架布局
系统 SHALL 提供一个带有左侧主导航栏（包含：工作台、新建任务、任务历史、标准库管理）和顶部用户信息栏的标准企业级布局。

### Requirement: 任务创建与上传
系统 SHALL 提供独立的“新建任务”页面。
#### Scenario: 批量上传 DWG 发起审查
- **WHEN** 用户进入新建任务页，将 100 个 DWG 文件拖入上传区域，选择适用的“审查标准”并点击“开始审查”
- **THEN** 系统显示上传进度，全部上传完成后，自动跳转至“任务列表”页，并显示该任务状态为“排队中/解析中”。

### Requirement: 任务列表与过滤
系统 SHALL 提供任务历史记录表格。
#### Scenario: 查找历史任务
- **WHEN** 用户在任务列表页选择日期范围和状态（如：已完成）
- **THEN** 表格自动过滤并展示符合条件的审查任务记录。

### Requirement: 审查结果展示（分栏视图）
系统 SHALL 提供左右分栏的审查结果详情页。
#### Scenario: 审查员查看具体文件的错误并准备在 CAD 中修改
- **WHEN** 用户在左侧面板点击某个文件名（如 `A区一层平面图.dwg`），并在右侧错误列表中点击某条错误的“在 CAD 中定位”按钮
- **THEN** 右侧面板仅显示该文件内的所有错别字和规范错误，且系统的剪贴板自动复制了该错误的 Handle ID（如 `2A5F`），并提示“已复制，请在 CAD 中使用快捷键定位”。

### Requirement: 标准库维护
系统 SHALL 提供标准库和字典的管理界面。
#### Scenario: 批量导入新标准
- **WHEN** 知识管理员在“标准库管理”页点击“导入 Excel”并上传整理好的规范文件
- **THEN** 系统解析 Excel 并将新标准条目追加至表格中，供后续的 AI 审查任务立即使用。

## MODIFIED Requirements
无（全新 Web 模块，无历史前端代码修改）。

## REMOVED Requirements
无。
